WHAT EACH COMPONENT DOES
================================================================================

FRONTEND PAGES:
  • GoogleOrderCheckout.tsx
    - Displays menu item from Google redirect
    - Collects customer information
    - Shows order type selection
    - Submits order for processing
    - Result: Order saved to database

  • OrderConfirmation.tsx
    - Shows order details and confirmation
    - Real-time order status tracking
    - Customer information display
    - Next steps guidance
    - Result: Customer can track order

MENU FEED SYSTEM:
  • googleMenuFeedManager.ts
    - Generates JSON/XML feeds from menu items
    - Includes action links for checkout
    - Handles feed caching
    - Updates feeds dynamically
    - Result: Menu available on Google

  • google-menu-feed edge function
    - Serves feeds to Google Merchant Center
    - Supports both JSON and XML formats
    - Caches with 1-hour TTL
    - Optimized for performance
    - Result: Google gets live menu feed

ORDER PROCESSING:
  • process-google-order edge function
    - Receives order from checkout page
    - Validates order data
    - Saves to database
    - Sends webhook to POS (HTTP)
    - Sends WebSocket notification (real-time)
    - Sends confirmation email
    - Result: Order in system + POS alerted

PAYMENT HANDLING:
  • google-payment-webhook edge function
    - Receives payment notification from Google
    - Updates order status
    - Notifies POS of payment
    - Sends customer confirmation
    - Result: Order confirmed + customer notified

REAL-TIME UPDATES:
  • useGoogleOrderNotifications hook
    - Subscribes to order updates via Supabase
    - Polls order status as fallback
    - Manages WebSocket connections
    - Broadcasts alerts
    - Result: Customer sees live status updates
